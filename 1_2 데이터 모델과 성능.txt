#1 성능 데이터 모델링 수행절차

정규화 → 용량산정 → 트랜잭션 유형 파악 → 반정규화 → 이력모델, PK/FK, 슈퍼타입/서브타입 조정 → 성능관점 데이터 모델 검증 

#2 성능 데이터 모델링

정규화는 기본적으로 중복데이터를 제거함으로써 조회성능 향상을 목적으로 함 

이력데이터의 경우는 컬럼을 추가 설게한다. 

#3 정규화

컬럼에 의한 반복적인 속성값을 갖는 형태 → 원자성 위배 → 1NF 대상

즉, 반복속성에 대해 인덱스를 생성하면 검색속도는 좋아지지만, 너무 많으면 입력, 수정, 삭제 성능이 저하되므로 1NF를 시행한다. 

컬럼 단위 중복은 1NF 대상이므로 1:M관계로 두 엔티티로 분리 

#4 반정규화

전체컬럼 중 자주 이용하는 집중화된 컬럼 존재 시 디스크의 I/O를 줄이기 위해 해당 컬럼들을 별도로 보다 좋은 반정규화 기법인 부분테이블 추가를 시행한다. 

이력테이블 추가 시 이력 특성을 고려하여 기능성 컬럼을 추가한다.

주의할 점은 FK 추가는 테이블간의 관계 연결시 생기는 자연스러운 현상이다. 

최근 변경된 값만 조회할 경우 과도한 조인으로 인해 성능 저하가 발생하므로 하나의 테이블로 토합하며 성능이 향상되지만 과도한 데이터가 한 테이블에 발생하여 용량이 커짐 

#5 대량 데이터

Partitioning: 논리적으로 하나, 물리적으로 여러개로 테이블 분리하여 성능이 향상, 관리방법 개선하도록 적용 

#6 데이터 변환

개별 테이블 모두 조회 → 하나의 테이블로 통합, PK체계 또는 일반속성 각 사건 구분자 부여 

#7 데이터 구조 - INDEX

값의 범위에 따라 일정하게 정렬 → 상속값으로 '=' 조건 가장 먼저 

즉, '='부분의 인덱스가 가장 앞쪽에 위치할 때 이용효율이 높음
